17. 부분 합 (Partial Sum)#

17.1 도입#

  • 부분 합(partial sum) 혹은 누적 합: 배열의 각 위치에 대해 배열의 시작부터 현재 위치까지의 원소의 합을 미리 구해 둔 배열
    • O(N), 수열의 길이에 선형으로 증가
  • partial sum(psum)을 미리 구해 두면, list의 특정 구간의 합(prefix sum)을 O(1)에 구할 수 있다.
    • psum[-1] = 0이라 가정할 때, list[a]부터 list[b]까지의 합은 psum[b] - psum[a-1] 이다.
  • 핵심: 계산한 부분 합 O(N)을 통한, 구간 합 빠르기 구하기, O(1)
  • 부분 합(psum) 은 첫 위치가 항상 A[0]으로 고정된, 구간 합(prefix sum) 이라고 할 수도 있다.

17.1.2 2차원으로의 확장#

  • 부분 합은 2차원 배열에서 A[y1, x1]에서 A[y2, x2]까지의 직사각형 구간의 합을 계산해야 할 때도 유용하다.

  • psum[y, x]는 (0, 0)을 왼쪽 위 칸, (y, x)를 오른쪽 아래 칸으로 갖는 직사각형 구간에 포함된 원소들의 합

  • 부분 합을 이용한 구간 합 구하기

    • 그림에서 굵은 선으로 표시된 구간의 합(prefix sum)
    • sum(y1, x1, y2, x2) = psum[y2, x2] - psum[y2, x1-1] - psum[y1-1, x2] + psum[y1-1, x1-1]

17.1.3 예제: 합이 0에 가장 가까운 구간#

  • 0 에 가장 가깝다는 말: psum[ ]의 두 값의 차이가 가장 적다는 말

    • 완전 탐색: O(N^2)

    • 부분 합 이용

      • 정렬: O(NlogN)

      • 얻은 부분 합을 이용해서,인접한 원소들 확인하는 것: O(N)

      • 총 결과: O(NlogN)

  • 배열에서 가장 가까운 두 값을 찾기 위한 방법

    • 이 배열을 정렬한 뒤, 인접한 원소들끼리 확인
    • Ex) 9, 100, 101, 300, 308 => (100, 101)

lst = [3, 5, 7, 9]
N = len(lst)
psum = [0] # 부분 합, 누적 합

for i in range(0, N):  # 부분 합 구하기
    psum.append(psum[i] + lst[i])  # [0, 3, 8, 15, 24]

# 0 <= leftIdx < rightIdx < N
def calPrefixSum(lIdx, rIdx): # leftIdx 부터 rightIdx 까지의 합
    global psum
    return psum[rIdx + 1] - psum[lIdx]

print(calPrefixSum(0, 0)) # 3
print(calPrefixSum(0, 1)) # 8
print(calPrefixSum(0, 3)) # 24
print(calPrefixSum(2, 3)) # 12
print(calPrefixSum(3, 3)) # 9

  • 초기에, psum = [0] 으로 초기화 해야, 이후 계산이 편해진다.

    • ‘psum[-1] = 0이라 가정’ 에 해당

  • 실제 PS에 부분 합, 구간 합 알고리즘을 쓸 때, 구간 인덱스가 0 or 1부터 인지 꼭 확인해야 한다.

  • 이 코드는 idx를 기준으로 계산하는 코드이다.

    • 0 <= leftIdx < rightIdx < N

  • 일반적인 구간 합 레퍼런스는 범위 정수를 기준으로 알려져 있다.

    • 이 경우에는 calPrefixSum(left, right) = psum[right - 1] - psum[left] 이다.
